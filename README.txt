小蔡同学文档工作室 - 顾客订单进度查询版

一、项目内容
1. public/index.html：前台接单网站。
2. public/admin.html：美观后台管理页面。
3. public/track.html：顾客订单进度查询页面。
4. server.js：Node.js + Express 后端。
5. package.json：运行配置。

二、前台功能
顾客提交需求后，会生成一个订单号。
顾客需要保存订单号，之后可以到 /track 查询订单进度。

三、顾客查询页
部署成功后，在网站地址后加：
/track

例如：
https://你的项目名.onrender.com/track

顾客需要输入：
1. 订单号
2. 提交订单时填写的联系方式

二者匹配后，才能查看订单进度。

四、后台入口
部署成功后，在网站地址后加：
/admin

例如：
https://你的项目名.onrender.com/admin

默认后台密码：
123456

正式使用建议在 Render 设置环境变量：
ADMIN_PASSWORD=你自己的密码

五、后台可以管理什么
1. 查看所有订单。
2. 修改订单状态：新订单、已联系、已报价、已收款、制作中、已完成、已取消。
3. 修改订单金额。
4. 修改顾客可见进度说明。
5. 写内部备注。
6. 删除订单。
7. 导出 CSV。
8. 复制顾客查询信息。

六、顾客能看到什么
顾客能看到：
1. 订单号
2. 服务类型
3. 截止时间
4. 订单金额
5. 当前状态
6. 进度时间线
7. 顾客可见进度说明
8. 自己提交的需求

顾客看不到：
1. 你的内部备注
2. 其他顾客订单
3. 后台管理页面

七、Render 部署更新方法
1. 解压本压缩包。
2. 进入 GitHub 仓库。
3. 删除或覆盖原来的 public 文件夹、server.js、package.json。
4. 上传本版本对应文件。
5. Commit changes。
6. Render 会自动重新部署。
7. 部署完成后打开 /admin 和 /track 测试。

八、重要说明
当前订单保存在 orders.json 文件里，适合前期测试和小规模接单。
Render 免费服务重启后文件可能丢失。
后期如果要长期稳定使用，建议升级为数据库版本。
